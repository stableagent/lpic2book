## swap، swapon، swapoff

قبلاً گفتیم که RAM مانند دروازه‌ای برای یک شهر است بنابراین بیش از حد مشغول است. همچنین ما برخی از تکنیک‌ها را معرفی کردیم که توسط سیستم عامل لینوکس برای مدیریت حافظه استفاده می‌شود. یکی از آن‌ها swap بود. swap حافظه شبیه‌سازی شده در دیسک است. اما چه مزایتی دارد و چگونه کار می‌کند؟ معمولاً swap استفاده می‌شود اگر کامپیوتر کمبود حافظه، در این وضعیت دو راه‌حل ممکن در دسترس هستند. اول جلوگیری از کاربر از اجرای برنامه یا برنامه‌هایی که حافظه بزرگتر از اندازه حافظه فیزیکی موجود را نیاز دارند و دوم، کرش کردن! واضح است که هیچ‌کدام از این راه‌حل‌ها قابل قبول نیست. swap به ما اجازه می‌دهد برنامه‌ها را اجرا کنیم و به آن‌ها بیش از آنچه واقعاً داریم حافظه اختصاص دهیم، با نوشتن داده‌ها روی دیسک سخت به طور موقت. با این حال هرچه این تکنیک تضمین عملکرد را نمی‌کند.

swap می‌تواند روی یک فایل (swap file) یا می‌تواند یک پارتیشن کامل دیسک (swap partition) باشد.

### swap file

بیایید یک فایل ایجاد کنیم و از آن به عنوان فضای swap استفاده کنیم:

```
root@server1:~# free -m
              total        used        free      shared  buff/cache   available
Mem:            971         623          67           3         280         159
Swap:          1020          10        1010
root@server1:~# swapon -s
Filename                Type        Size    Used    Priority
/dev/sda5                                  partition    1045500    10976    -1
```

دستور swapon -s به ما خلاصه‌ای از فضاهای swap تخصیص داده شده می‌دهد.

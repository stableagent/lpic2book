# 208.1. پیاده‌سازی یک سرور وب

## **208.1 Implementing a webserver**

**وزن:**4

**توضیحات:**داوطلبان باید بتوانند یک سرور وب نصب و پیکربندی کنند. این هدف شامل مانیتورینگ بار و عملکرد سرور، محدود کردن دسترسی کاربر کلاینت، پیکربندی پشتیبانی برای زبان‌های اسکریپتنویسی به عنوان ماژول و تنظیم احراز هویت کاربر کلاینت است. همچنین شامل پیکربندی گزینه‌های سرور برای محدود کردن استفاده از منابع است. داوطلبان باید بتوانند سرور وب را پیکربندی کنند تا از virtual hosts استفاده کند و دسترسی فایل را سفارشی کند.

**نواحی کلیدی دانش:**

* فایل‌های پیکربندی Apache 2.4، اصطلاحات و ابزارها
* پیکربندی و محتویات فایل‌های log Apache
* روش‌ها و فایل‌های محدود کردن دسترسی
* پیکربندی mod\_perl و PHP
* فایل‌ها و ابزارهای احراز هویت کاربر کلاینت
* پیکربندی حداکثر درخواست‌ها، حداقل و حداکثر سرورها و کلاینت‌ها
* پیاده‌سازی virtual host Apache 2.4 (با و بدون آدرس‌های IP اختصاصی)
* استفاده از عبارات redirect در فایل‌های پیکربندی Apache برای سفارشی کردن دسترسی فایل

**اصطلاحات و ابزارها:**

* logهای دسترسی و logهای خطا
* .htaccess
* httpd.conf
* mod\_auth\_basic، mod\_authz\_host و mod\_access\_compat
* htpasswd
* AuthUserFile، AuthGroupFile
* apachectl، apache2ctl
* httpd، apache2

## سرور وب چیست؟

یک سرور وب برنامه‌ای است که از HTTP (پروتکل انتقال Hypertext) استفاده می‌کند تا فایل‌هایی که صفحات وب را تشکیل می‌دهند را به کاربران، در پاسخ به درخواست‌های آن‌ها، که توسط کلاینت‌های HTTP کامپیوترهای آن‌ها ارسال شده‌اند، خدمت دهد. کامپیوترها و دستگاه‌های اختصاصی نیز ممکن است به عنوان سرورهای وب اشاره شوند.

![](.gitbook/assets/apache-webserver.jpg)

برخی ملاحظات در انتخاب یک سرور وب شامل این است که چقدر خوب با سیستم عامل و سایر سرورها کار می‌کند، توانایی آن در مدیریت برنامه‌نویسی سمت سرور، ویژگی‌های امنیتی، و ابزارهای انتشار خاص، موتور جستجو و ساخت سایت که با آن می‌آیند، وجود دارد.

سرورهای وب پیشرو شامل Apache (رایج‌ترین سرور وب نصب شده)، سرور اطلاعات اینترنت مایکروسافت (IIS) و nginx (تلفظ می‌شود engine X) از NGNIX هستند. سایر سرورهای وب شامل سرور NetWare ناول، سرور وب گوگل (GWS) و خانواده سرورهای Domino آی‌بی‌ام هستند.

## Apache![](.gitbook/assets/apache-logo.png)

پروژه سرور HTTP Apache تلاشی است برای توسعه و حفظ یک سرور HTTP متن‌باز برای سیستم‌های عامل مدرن شامل UNIX و ویندوز. هدف این پروژه فراهم کردن یک سرور امن، کارا و گسترش‌پذیر است که خدمات HTTP را در همگامی با استانداردهای فعلی HTTP فراهم می‌کند.

سرور HTTP Apache ("httpd") در سال 1995 راه‌اندازی شد و از آوریل 1996 محبوب‌ترین سرور وب در اینترنت بوده است. این پروژه در فوریه 2015 بیستمین تولد خود را به عنوان یک پروژه جشن گرفت. سرور HTTP Apache یک پروژه از بنیاد نرم‌افزار Apache است. نسخه فعلی apache نسخه 2.0 است.

## httpd.conf

فایل‌های پیکربندی Apache به طور متفاوت در توزیع‌های مختلف ذخیره می‌شوند، اما عمدتاً چیز یکسانی هستند. یک فایل پیکربندی استاندارد وجود دارد که ممکن است به بسیاری از قطعات شکسته شده باشد.

| apache                       | Redhat / CentOS | Debian / Ubuntu |
| ---------------------------- | --------------- | --------------- |
| نام بسته                 | httpd           | apache2         |
| محل فایل‌های پیکربندی | /etc/httpd      | /etc/apache2    |
